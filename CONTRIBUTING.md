## Getting Started

### 1. Fork and Clone

```bash
# Fork on GitHub, then clone your fork
git clone https://github.com/YOUR_USERNAME/nexus.git
cd nexus
```

### 2. Set Up Development Environment

```bash
# Create virtual environment
python3.11 -m venv venv
source venv/bin/activate

# Install dependencies (development mode)
pip install -e ".[dev]"

# Copy environment template
cp .env.example .env
# Edit .env with your local configuration
```

### 3. Start Infrastructure

```bash
# Start PostgreSQL, Redis, MinIO via Docker
docker-compose up -d

# Run database migrations
alembic upgrade head

# Verify database
alembic current
```

### 4. Run the Application

```bash
# Terminal 1: API server with hot reload
uvicorn nexus.api.main:app --reload --host 0.0.0.0 --port 8000

# Terminal 2: Celery worker
celery -A nexus.workers.app worker --loglevel=info

# Terminal 3: Celery beat scheduler (for recurring tasks)
celery -A nexus.workers.app beat --loglevel=info

# Terminal 4 (optional): Next.js web dashboard
cd web && npm install && npm run dev
```

### 5. Run Tests

```bash
# Full test suite (should show 163 passing)
pytest

# With coverage report
pytest --cov=nexus --cov-report=html

# Open coverage report
open htmlcov/index.html
```

### 6. Create a Feature Branch

```bash
git checkout -b feature/your-feature-name
```

## Development Workflow

### Running Tests

```bash
# Full suite (163 tests)
pytest

# Specific test file
pytest tests/test_scheduling.py -v

# Specific test function
pytest tests/test_scheduling.py::test_parse_nl_date_tomorrow -v

# With coverage
pytest --cov=nexus --cov-report=html

# Watch mode (requires pytest-watch)
pytest-watch
```

**Test Organization:**

- `tests/conftest.py` вҖ” Pytest fixtures (db_session, test_user, test_client)
- `tests/test_*.py` вҖ” Test modules mirror `src/nexus/` structure
- Integration tests use `test_client` fixture (FastAPI TestClient)
- Unit tests mock external dependencies (OpenAI, Twilio, Tesseract)

### Code Quality

Before submitting a PR, ensure your code passes all checks:

```bash
# Lint with ruff
ruff check src/ tests/

# Type check with mypy
mypy src/

# Format with black
black src/ tests/

# Check formatting without modifying files
black --check src/ tests/

# All checks at once
black src/ tests/ && mypy src/ && ruff check src/ tests/
```

### Database Migrations

When modifying models in `src/nexus/models/`:

```bash
# 1. Generate migration (autogenerate from model changes)
alembic revision --autogenerate -m "Add vendor_aliases table"

# 2. Review the generated migration in alembic/versions/
#    Ensure it matches your intent (drop/add columns, indexes, constraints)

# 3. Apply migration
alembic upgrade head

# 4. Verify current migration
alembic current

# 5. Test rollback (optional but recommended)
alembic downgrade -1
alembic upgrade head
```

**Migration Best Practices:**

- Always review autogenerated migrations before committing
- Add data migrations separately (don't mix schema + data changes)
- Test migrations on a copy of production data before deploying
- Never edit migrations after they've been pushed to `main`

### Layer Responsibilities

| Layer | Directory | Purpose | Examples |
|-------|-----------|---------|----------|
| **Models** | `src/nexus/models/` | SQLAlchemy ORM, database schema | `User`, `Task`, `Transaction`, `Note` |
| **Services** | `src/nexus/services/` | Business logic, ML, external APIs | `forecasting.py`, `scheduling.py`, `voice.py` |
| **Routers** | `src/nexus/api/routers/` | HTTP endpoints, request/response | `tasks.py`, `finance.py`, `research.py` |
| **Utils** | `src/nexus/utils/` | Reusable utilities, no state | `categorizer.py`, `vendors.py`, `security.py` |
| **Workers** | `src/nexus/workers/` | Background tasks (Celery) | `tasks.py` (recurring, backups) |
| **CLI** | `src/nexus/cli/` | Click commands | `main.py` (all CLI commands) |

### Design Patterns

#### 1. Pluggable Graceful Degradation

External services (OpenAI, Twilio, Tesseract) use circuit breakers and fallbacks:

```python
from nexus.utils.resilience import with_circuit_breaker

@with_circuit_breaker(max_failures=3, timeout=60)
async def call_external_api():
    # Primary implementation
    pass
```

See `src/nexus/utils/resilience.py` for implementation.

#### 2. Service Layer Pattern

Business logic lives in services, not routers:

```python
# вң… GOOD: Router delegates to service
@router.post("/tasks")
async def create_task(task_data: TaskCreate, db: AsyncSession):
    conflicts = await detect_conflicts(task_data.due_date, db)
    if conflicts:
        raise HTTPException(409, detail="Scheduling conflict")
    return await create_task_service(task_data, db)

# вқҢ BAD: Business logic in router
@router.post("/tasks")
async def create_task(task_data: TaskCreate, db: AsyncSession):
    # Complex logic here...
```

#### 3. Dependency Injection

Use FastAPI's dependency injection for database sessions, authentication:

```python
from nexus.api.dependencies import get_current_user, get_db

@router.get("/tasks")
async def list_tasks(
    db: AsyncSession = Depends(get_db),
    user: User = Depends(get_current_user)
):
    return await get_user_tasks(user.id, db)
```

#### 4. Async/Await Everywhere

All I/O operations use `async`/`await`:

```python
# Database
async with db.begin():
    result = await db.execute(select(Task).where(Task.user_id == user_id))

# HTTP clients
async with httpx.AsyncClient() as client:
    response = await client.get(url)

# File I/O (when possible)
async with aiofiles.open(path, "r") as f:
    content = await f.read()
```

```python
from sqlalchemy import Column, Integer, String, ForeignKey
from sqlalchemy.orm import relationship
from .base import Base

class Example(Base):
    __tablename__ = "examples"
```

#### B. Service (if needed)

Create service in `src/nexus/services/`:

```python
# src/nexus/services/example.py
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select
from nexus.models.example import Example

async def create_example(name: str, user_id: int, db: AsyncSession) -> Example:
    """Create a new example.
    
    Args:
        name: Example name
        user_id: Owner's user ID
        db: Database session
        
    Returns:
        Created example instance
    """
    example = Example(name=name, user_id=user_id)
    db.add(example)
    await db.commit()
    await db.refresh(example)
    return example
```

#### C. Router

Add endpoints in `src/nexus/api/routers/`:

```python
# src/nexus/api/routers/example.py

from sqlalchemy.ext.asyncio import AsyncSession
from nexus.api.dependencies import get_db, get_current_user
from nexus.models.user import User

router = APIRouter(prefix="/examples", tags=["examples"])

@router.post("", status_code=201)

    name: str,
    db: AsyncSession = Depends(get_db),
    user: User = Depends(get_current_user)

    return {"id": example.id, "name": example.name}
```

## Testing Guidelines

### Test Structure

```python
# tests/test_scheduling.py

import pytest
from datetime import datetime, timezone
from nexus.services.scheduling import parse_nl_date, detect_conflicts

# 1. Unit tests for pure functions
def test_parse_nl_date_tomorrow():
    """Test natural language date parsing for 'tomorrow'."""
    result = parse_nl_date("tomorrow 3pm")
    assert result is not None
    assert result.hour == 15

# 2. Integration tests with database
@pytest.mark.asyncio
async def test_detect_conflicts(db_session, test_user):
    """Test conflict detection with overlapping tasks."""
    # Create test task
    task1 = Task(
        title="Meeting",
        due_date=datetime(2026, 7, 13, 9, 0, tzinfo=timezone.utc),
        user_id=test_user.id
    )
    db_session.add(task1)
    await db_session.commit()
    
    # Check for conflicts
    conflicts = await detect_conflicts(
        test_user.id,
        datetime(2026, 7, 13, 8, 30, tzinfo=timezone.utc),
        db_session

    assert len(conflicts) > 0

# 3. API integration tests
@pytest.mark.asyncio
async def test_suggest_time_endpoint(test_client, auth_headers):
```

## Coding Standards

### Python Style

- Follow **PEP 8**
- Use **type hints** for all function signatures
- Maximum line length: **88 characters** (Black default)
- Use **async/await** for all I/O operations
- Prefer **composition over inheritance**
- Use **dataclasses** or **Pydantic models** for structured data

### Example Function

```python
from typing import Optional
from datetime import datetime
from sqlalchemy.ext.asyncio import AsyncSession
from nexus.models.task import Task

async def get_task_by_id(
    task_id: int,
    user_id: int,
    db: AsyncSession
) -> Optional[Task]:
    """Retrieve a task by ID if it belongs to the user.
    
    Args:
        task_id: Task primary key
        user_id: Owner's user ID
        db: Async database session
        
    Returns:
        Task instance if found and owned by user, else None
        
    Raises:
        SQLAlchemyError: On database errors
    """
    result = await db.execute(
        select(Task)
        .where(Task.id == task_id)
        .where(Task.user_id == user_id)
    )
    return result.scalar_one_or_none()
```

## Security Guidelines

### What NOT to Commit

- вқҢ API keys or secrets (OpenAI, Twilio, etc.)
- вқҢ `.env` files (use `.env.example` instead)
- вқҢ Personal data or test user credentials
- вқҢ Database backups or dumps
- вқҢ Private keys or certificates

### What TO Do

- вң… Use environment variables for all secrets
- вң… Add new secrets to `.env.example` (with placeholder values)
- вң… Hash passwords with bcrypt (never plaintext)
- вң… Encrypt sensitive fields (use `EncryptedType` for DB columns)
- вң… Validate user input (Pydantic models enforce schemas)
- вң… Use parameterized queries (SQLAlchemy ORM prevents SQL injection)
- вң… Rate limit external-facing endpoints (`utils/ratelimit.py`)

### Reporting Vulnerabilities

**Do NOT** open public issues for security vulnerabilities.

Instead, email **calvinbrady8@gmail.com** with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

See [SECURITY.md](SECURITY.md) for the full disclosure policy.

## Common Tasks

### Add a New API Endpoint

1. Create service function in `src/nexus/services/`
2. Add route in `src/nexus/api/routers/`
3. Register router in `src/nexus/api/main.py` (if new router)
4. Write integration test in `tests/test_<router>.py`
5. Add CLI command in `src/nexus/cli/main.py`

### Add a New Database Table

1. Create model in `src/nexus/models/`
2. Import model in `src/nexus/models/__init__.py`
3. Generate migration: `alembic revision --autogenerate -m "Description"`
4. Review migration file in `alembic/versions/`
5. Apply migration: `alembic upgrade head`
6. Add relationships to existing models (if needed)

### Add a New Celery Task

1. Define task in `src/nexus/workers/tasks.py`
2. Configure schedule in `src/nexus/workers/app.py` (if periodic)
3. Test manually: `celery -A nexus.workers.app call nexus.workers.tasks.your_task`
4. Monitor logs: `celery -A nexus.workers.app worker --loglevel=debug`

### Add External Service Integration

1. Add circuit breaker wrapper (see `utils/resilience.py`)
2. Store credentials in `.env` + `config.py`
3. Create service module in `services/`
4. Mock service in tests (use `pytest-mock` or `unittest.mock`)
5. Add graceful fallback behavior
